# Architecture Overview

Eden UI is built with a focus on developer experience, performance, and maintainability.

## Technology Stack

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Framework** | Svelte 5 | Component runtime with runes |
| **Styling** | Tailwind CSS 4 | Utility-first CSS |
| **Variants** | tailwind-variants | Type-safe variant composition |
| **Build** | Vite + SvelteKit | Development and packaging |
| **Types** | TypeScript | Full type safety |

## Component Architecture

### Standard Component Structure

Every Eden UI component follows this pattern:

```
component-name/
├── ComponentName.svelte    # Main component
├── theme.ts                # tailwind-variants styling
├── index.ts                # Exports
└── SubComponent.svelte     # Optional sub-components
```

### Theme Files

Theme files use `tailwind-variants` for type-safe, composable styling:

```typescript title="src/lib/buttons/theme.ts"
import { tv, type VariantProps } from 'tailwind-variants';

export type ButtonVariants = VariantProps<typeof button>;

export const button = tv({
  base: 'inline-flex items-center justify-center font-medium transition-colors focus:outline-none focus:ring-2',

  variants: {
    color: {
      primary: 'bg-primary-600 text-white hover:bg-primary-700',
      secondary: 'bg-gray-200 text-gray-900 hover:bg-gray-300',
      // ... 18 more colors
    },
    size: {
      xs: 'px-2 py-1 text-xs',
      sm: 'px-3 py-1.5 text-sm',
      md: 'px-4 py-2 text-sm',
      lg: 'px-5 py-2.5 text-base',
      xl: 'px-6 py-3 text-lg',
    },
    outline: {
      true: 'bg-transparent border',
    },
    pill: {
      true: 'rounded-full',
      false: 'rounded-lg',
    },
    shadow: {
      true: 'shadow-lg',
    },
  },

  compoundVariants: [
    // Outline + Color combinations
    {
      outline: true,
      color: 'primary',
      class: 'border-primary-600 text-primary-600 hover:bg-primary-50',
    },
    // ... compound variants for all colors
  ],

  defaultVariants: {
    size: 'md',
    color: 'primary',
    pill: false,
  },
});
```

### Component Files

Components use Svelte 5 runes and compose with their theme:

```svelte title="src/lib/buttons/Button.svelte"
<script lang="ts">
  import { button, type ButtonVariants } from './theme';
  import type { Snippet } from 'svelte';

  interface Props extends ButtonVariants {
    href?: string;
    disabled?: boolean;
    class?: string;
    children?: Snippet;
  }

  let {
    color = 'primary',
    size = 'md',
    outline = false,
    pill = false,
    shadow = false,
    href,
    disabled = false,
    class: className = '',
    children,
    ...restProps
  }: Props = $props();

  const classes = $derived(
    button({ color, size, outline, pill, shadow, class: className })
  );
</script>

{#if href}
  <a {href} class={classes} {...restProps}>
    {@render children?.()}
  </a>
{:else}
  <button class={classes} {disabled} {...restProps}>
    {@render children?.()}
  </button>
{/if}
```

## Design Decisions

### Why tailwind-variants?

1. **Type Safety**: Full TypeScript inference for all variant combinations
2. **Composability**: Extend base themes for custom variants
3. **Performance**: Compiles to static class strings at build time
4. **Readability**: Clear variant definitions in one place

### Why Svelte 5 Runes?

1. **Explicit Reactivity**: `$state`, `$derived`, `$effect` are explicit
2. **Better TypeScript**: Props type inference is automatic
3. **Snippets**: More flexible than slots for composition
4. **Performance**: Fine-grained reactivity without overhead

### Color System (20 Colors)

We support 20 colors to cover:
- **Brand Colors**: primary, secondary
- **Semantic Colors**: red (error), green (success), yellow (warning), blue (info)
- **Full Spectrum**: All Tailwind color families for design flexibility

<Aside type="note">
  Every component that supports colors implements all 20 variants consistently.
</Aside>

## Bundle Architecture

Eden UI is published as a single package with tree-shaking support:

```
dist/
├── index.js            # Main ESM bundle
├── index.d.ts          # TypeScript definitions
├── accordion/          # Individual component chunks
├── buttons/
├── ...
└── styles.css          # Optional base styles
```

Importing specific components only bundles what you use:

```svelte
// Only Button and Card are included in your bundle
import { Button, Card } from 'eden-ui';
```
