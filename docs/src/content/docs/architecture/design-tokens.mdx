# Design Tokens

Design tokens are the foundational values that define Eden UI's visual language. They ensure consistency across all components.

## Color Tokens

Eden UI uses Tailwind CSS's default color palette, extended with semantic naming:

### Primary Palette

| Token | Light Mode | Dark Mode | Usage |
|-------|------------|-----------|-------|
| `primary-50` | `#eff6ff` | - | Subtle backgrounds |
| `primary-100` | `#dbeafe` | - | Light backgrounds |
| `primary-500` | `#3b82f6` | - | Default state |
| `primary-600` | `#2563eb` | - | Hover state |
| `primary-700` | `#1d4ed8` | - | Active state |

### Semantic Colors

| Purpose | Color | Token |
|---------|-------|-------|
| Success | Green | `green-500`, `emerald-500` |
| Warning | Yellow/Amber | `yellow-500`, `amber-500` |
| Error | Red | `red-500` |
| Info | Blue | `blue-500`, `sky-500` |

## Spacing Scale

Based on Tailwind's default 4px base unit:

| Token | Value | Usage |
|-------|-------|-------|
| `0` | 0px | No spacing |
| `1` | 4px | Tight spacing |
| `2` | 8px | Compact spacing |
| `3` | 12px | Default inline |
| `4` | 16px | Default block |
| `6` | 24px | Section spacing |
| `8` | 32px | Large spacing |

## Typography Scale

| Token | Size | Line Height | Usage |
|-------|------|-------------|-------|
| `text-xs` | 12px | 16px | Labels, captions |
| `text-sm` | 14px | 20px | Secondary text |
| `text-base` | 16px | 24px | Body text |
| `text-lg` | 18px | 28px | Large body |
| `text-xl` | 20px | 28px | Subheadings |
| `text-2xl` | 24px | 32px | Headings |

## Border Radius

| Token | Value | Usage |
|-------|-------|-------|
| `rounded-none` | 0 | Sharp corners |
| `rounded-sm` | 2px | Subtle rounding |
| `rounded` | 4px | Default |
| `rounded-md` | 6px | Medium rounding |
| `rounded-lg` | 8px | Cards, modals |
| `rounded-xl` | 12px | Large cards |
| `rounded-full` | 9999px | Pills, avatars |

## Shadow Scale

| Token | Usage |
|-------|-------|
| `shadow-sm` | Subtle elevation |
| `shadow` | Default cards |
| `shadow-md` | Dropdowns |
| `shadow-lg` | Modals |
| `shadow-xl` | Popovers |
| `shadow-2xl` | Dialogs |

## Using Tokens in Components

Tokens are applied via tailwind-variants:

```typescript
import { tv } from 'tailwind-variants';

export const card = tv({
  base: 'rounded-lg shadow-md p-4',
  variants: {
    size: {
      sm: 'p-3 rounded-md',
      md: 'p-4 rounded-lg',
      lg: 'p-6 rounded-xl',
    }
  }
});
```

<Aside type="tip">
  Use semantic tokens (like `primary`, `success`, `danger`) rather than raw color values for better maintainability and theme support.
</Aside>

## Customizing Tokens

Override tokens in your `tailwind.config.js`:

```js title="tailwind.config.js"
export default {
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#your-value',
          // ... custom primary scale
        }
      },
      spacing: {
        '18': '4.5rem',  // Custom spacing value
      }
    }
  }
}
```
