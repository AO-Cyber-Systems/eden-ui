# Button

Buttons trigger actions or navigation. Eden UI buttons support extensive customization while maintaining accessibility.

## Live Demo

<WorkshopEmbed story="buttons/Button" height="300px" />

## Import

```svelte
import { Button } from 'eden-ui';
```

## Basic Usage

```svelte
<Button>Default Button</Button>
<Button color="primary">Primary</Button>
<Button color="green">Success</Button>
<Button color="red">Danger</Button>
```

## With Icons

```svelte
<script>
  import { Button } from 'eden-ui';
  import { PlusIcon, ArrowRightIcon } from 'lucide-svelte';
</script>

<!-- Icon before text -->
<Button>
  <PlusIcon class="w-4 h-4 mr-2" />
  Add Item
</Button>

<!-- Icon after text -->
<Button color="primary">
  Continue
  <ArrowRightIcon class="w-4 h-4 ml-2" />
</Button>

<!-- Icon only -->
<Button color="gray" aria-label="Add">
  <PlusIcon class="w-5 h-5" />
</Button>
```

<Aside type="tip">
  When using icon-only buttons, always include an `aria-label` for accessibility.
</Aside>

## Loading State

```svelte
<script>
  import { Button, Spinner } from 'eden-ui';

  let loading = $state(false);

  async function handleClick() {
    loading = true;
    await doSomething();
    loading = false;
  }
</script>

<Button onclick={handleClick} disabled={loading}>
  {#if loading}
    <Spinner size="sm" class="mr-2" />
    Loading...
  {:else}
    Submit
  {/if}
</Button>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `color` | `'primary' \| 'secondary' \| 'gray' \| ...` | `'primary'` | Button color variant |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Button size |
| `outline` | `boolean` | `false` | Use outline style |
| `pill` | `boolean` | `false` | Use fully rounded corners |
| `shadow` | `boolean` | `false` | Add drop shadow |
| `disabled` | `boolean` | `false` | Disable the button |
| `href` | `string` | - | Render as `<a>` tag |
| `target` | `string` | - | Link target (when href is set) |
| `class` | `string` | - | Additional CSS classes |

## Accessibility

- Buttons use native `<button>` elements (or `<a>` when `href` is provided)
- Focus states are clearly visible with ring indicators
- Disabled buttons are properly marked with `aria-disabled`
- Color contrast meets WCAG AA standards
